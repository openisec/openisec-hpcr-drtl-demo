## HPCR-DRTL Framework

AI-powered decision support platform for enterprise governance by using HPCR-DRTL = History / Pro / Con / Recommendation / Decision / Reason / Target / Log.

| Letter | Label | Description (EN) | 説明 (JP) |
|--------|-------|------------------|-----------|
| H | History — 経緯・文脈 | Organizes keywords chronologically and provides background context | キーワードを時系列で整理し、経緯・文脈を提供 |
| P | Pro — メリット | Concisely presents the advantages of each option | 選択肢ごとの優位点を簡潔に提示 |
| C | Con — デメリット | Concisely presents the risks and challenges of each option | 選択肢ごとのリスク・課題を簡潔に提示 |
| R | Recommendation — 推奨事項 | Objective recommendation generated by AI | AIによる客観的な推奨内容 |
| D | Decision — あなたの決定 | Records the user's own final decision | ユーザー自身の最終決定を記録 |
| R | Reason — 判断理由 | Records the rationale for adopting, modifying, or rejecting the AI recommendation | AI推奨を採用・修正・見送りした判断理由を記録 |
| T | Target — 実施ターゲット日 | Sets and manages the planned implementation date of the decision | 決定の実施予定日を設定・管理 |
| L | Log — 決定を記録 | Saves the decision, rationale, and target date to the database | 決定内容・判断理由・ターゲット日をDBに保存 |

The AI (Gemini) handles H, P, C, R — the human retains ownership of D, R, T, L.

---

## Key Features

- 🤖 AI analysis (Gemini 2.5 Flash) with risk score and confidence evaluation
- 📏 Dynamic character limits based on query length
- 📋 Flexible Pro/Con bullet points that adjust based on the number of options
- 💡 Records the human's rationale (Reason) for accepting, modifying, or rejecting AI recommendations
- 📅 Target date management for implementation planning
- 🔐 Bearer Token authentication, input guardrails, and PII masking

| CI/CD | GitHub Actions / Snyk / pytest |

---

## Security

- OWASP LLM Top 10:2025 compliant input guardrails
- Snyk SAST / SCA scanning in CI/CD
- Password hashing with Argon2id
- All secrets managed via environment variables (never committed to repository)
- SQL injection protection via SQLAlchemy ORM
- PII masking before AI processing

## Self-Hosting (Local Development)

### Prerequisites

- Docker Desktop
- Google Gemini API Key ([Get one free](https://aistudio.google.com/apikey))

### Quick Start

```bash
# 1. Clone the repository
git clone https://github.com/openisec/openisec-hpcr-drtl-demo.git
cd openisec-hpcr-drtl-demo

# 2. Configure API environment
cp apps/api/.env.example apps/api/.env
# Edit apps/api/.env and set your GEMINI_API_KEY

# 3. Start all services
docker compose up -d

# 4. Open in browser
# http://localhost:3000
```
